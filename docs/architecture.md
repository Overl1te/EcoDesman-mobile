# Архитектура EcoNizhny Mobile

## Основная идея

Клиент остаётся тонким: он получает данные с Django API, хранит только токены и минимальное состояние интерфейса.

## Слои

```text
lib/
  app/              # bootstrap, router, shell
  core/             # конфиг, сеть, хранение токенов, утилиты
  features/
    auth/           # splash, login, восстановление сессии
    feed/           # лента, деталка поста, пагинация
    map/            # будущая вкладка
    events/         # будущая вкладка
    profile/        # профиль и выход
  shared/widgets/   # общие состояния empty/error
```

## Почему это удобно для роста

- auth и feed уже развязаны через репозитории;
- экран не ходит в сеть напрямую, всё идёт через controllers/repositories;
- карту, мероприятия и комментарии можно добавлять отдельными фичами без переписывания текущих потоков.
